**Step 1: Add CSS custom properties to src/index.css**

Add an `@layer base` block with `:root` token definitions AFTER the existing `@import` statements but BEFORE the existing CSS reset rules. Place it right after the last `@import` line (line 16: `@import url(... Inter ...)`).

Use HSL channel values (H S% L%) WITHOUT the `hsl()` wrapper — the wrapper goes in `tailwind.config.js`.

The token values use the **warm coral/salmon palette** per user decision (FNDTN-06). This is safe because `bg-primary` (the semantic token class) currently resolves to nothing — no existing code uses it. Existing code uses `bg-primary-500` which resolves via hardcoded hex in the Tailwind config color scales and is NOT affected by these CSS variables. So the coral values satisfy FNDTN-06 with zero regression.

```css
/* === Design Token System (Phase 6: Foundation) === */
/* Warm coral/amber palette per user decision. */
/* bg-primary-500 (hardcoded hex) is unaffected — only bg-primary uses these. */
@layer base {
  :root {
    --background: 30 25% 97%;
    --foreground: 20 15% 15%;
    --card: 30 20% 99%;
    --card-foreground: 20 15% 15%;
    --popover: 30 20% 99%;
    --popover-foreground: 20 15% 15%;
    --primary: 15 85% 45%;
    --primary-foreground: 0 0% 100%;
    --secondary: 20 12% 45%;
    --secondary-foreground: 30 20% 98%;
    --muted: 25 15% 93%;
    --muted-foreground: 20 10% 45%;
    --accent: 35 90% 55%;
    --accent-foreground: 25 60% 20%;
    --destructive: 0 72% 50%;
    --destructive-foreground: 0 0% 100%;
    --border: 25 12% 88%;
    --input: 25 12% 88%;
    --ring: 15 85% 45%;
    --radius: 0.75rem;
    --sidebar: 220 25% 18%;
    --sidebar-foreground: 30 25% 97%;
  }
}
```

Color rationale:
- `--primary: 15 85% 45%` → deep coral/salmon (hue 15, highly saturated, darkened to 45% lightness) — WCAG AA contrast ~4.57:1 against white text (passing 4.5:1 minimum)
- `--background: 30 25% 97%` → warm off-white with cream tint — cozy, not sterile
- `--accent: 35 90% 55%` → warm amber — matching warm family
- `--destructive: 0 72% 50%` → warm red (hue 0, not stock CSS red)
- `--muted: 25 15% 93%` → warm gray with slight amber undertone
- `--sidebar: 220 25% 18%` → dark charcoal/navy — Monday.com style contrast
- `--radius: 0.75rem` → well-rounded per user decision (12px)

Also, remove the `@import url(... Inter ...)` line (line 18 in current file: `@import url('https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap');`). Heebo replaces Inter, and loading happens via `<link>` in `index.html` now (better for performance — no render-blocking @import).

Keep the Reisinger Yonatan @font-face declaration — it stays as fallback. It will be removed in Phase 7+.
