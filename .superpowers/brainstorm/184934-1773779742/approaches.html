A: FullCalendar Library

Library: @fullcalendar/react — the industry standard

  • Built-in month / week / day / list views with smooth transitions
  • Native RTL support + Hebrew locale out of the box
  • Drag-and-drop rescheduling (drag event to new date/time)
  • Event resize (drag bottom edge to change duration)
  • Color-coded by orchestra type with left border accent
  • Today highlight, multi-day events, overflow "+N more"
  • Fully customizable event rendering with your own components
  • ~40KB gzipped — well maintained, used by Google, Microsoft
✦ Recommended — replaces ~1400 lines of custom calendar code with a battle-tested library

B: Timeline Cards View

Approach: Day-grouped card list as the primary view

  • Rehearsals grouped by day with sticky date headers
  • Rich cards — orchestra name, time, location, attendance bar, status badge
  • Color-coded left border by orchestra type (indigo/pink)
  • Inline attendance progress bar with percentage
  • Calendar toggle still available for spatial overview
  • Much more info at a glance vs. tiny calendar cells
  • Works great on mobile — no cramped grid cells
  • Built with HeroUI Card + custom layout (no heavy lib needed)
✦ Best for daily workflow — see everything happening with full detail

My Recommendation

Use both together. FullCalendar as the calendar view (replacing ~1400 lines of custom code), and Timeline Cards as the list view (replacing the plain table). Toggle between them just like today. The calendar gives spatial overview ("when is everything?"), the cards give operational detail ("what's next and what do I need to do?").
